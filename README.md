# Oda-Fragment-pull

Downstream data repository for the **Oda Fragment** Discord bot (a Cephalon-style Warframe assistant powered by **Google Gemma 4**). This repo holds the AI-readable `.txt` knowledge bundles that the bot retrieves from at runtime.

> **This repository is auto-populated.** Do not hand-edit files under `data/`. They are overwritten on every publish run from the upstream pipeline at [`aidenlong04/warframe-item-pull`](https://github.com/aidenlong04/warframe-item-pull).

## What lives here

| Path | Purpose |
|------|---------|
| `data/warframe-data-*.txt` | AI-readable knowledge bundles (≤3 MB each, ~14 MB total) consumed by the Oda Fragment bot's retriever |
| `data/MANIFEST.md` | Auto-generated manifest: source commit, generation timestamp, workflow run URL, and per-file sizes |
| `README.md` | This file — repository overview for humans and AI agents |

The `.txt` bundles are produced upstream from a four-stage pipeline (`pull` → `extract-wiki` → `transform` → `organize`) and are tuned for Gemma 4's smaller context window (~128K tokens) and stricter instruction-following.

## Upstream source

All content is generated by [`aidenlong04/warframe-item-pull`](https://github.com/aidenlong04/warframe-item-pull):

1. `pull.js` — Fetches raw JSON from community APIs and npm packages (`@wfcd/items`, `@wfcd/patchlogs`, `@wfcd/relics`, `warframe-worldstate-data`, `warframe-public-export-plus`, `warframe-riven-info`, `drops.warframestat.us`).
2. `extract-wiki.py` — Extracts the HuggingFace `LLukas22/wf-wiki` parquet dataset into 10 curated category JSONs plus a wiki-enrichment file.
3. `transform.js` — Converts all JSON into structured Markdown under `docs/`.
4. `organize-docs.js` — Bundles Markdown into 15 themed `.txt` files (≤3 MB each).

The upstream workflow `.github/workflows/publish-data.yml` then checks out this repo, replaces `data/warframe-data-*.txt`, regenerates `data/MANIFEST.md`, and commits with message `data: refresh warframe data bundles (source <sha>)`.

## Update cadence

The upstream publish workflow runs automatically:

- **Weekly schedule** — every Monday at `06:00 UTC` (cron `0 6 * * 1`).
- **On push to `main`** in `warframe-item-pull` when any of `pull.js`, `transform.js`, `organize-docs.js`, `extract-wiki.py`, or `.github/workflows/publish-data.yml` changes.
- **On manual dispatch** (`workflow_dispatch`).

Each publish run rewrites `data/warframe-data-*.txt`, removing any files no longer produced upstream so deletions propagate, and refreshes `data/MANIFEST.md` with the new source commit and timestamp.

## Conventions (for AI consumers and agents)

The bundles are produced for Gemma 4 and follow these rules end-to-end:

- **ASCII-only output.** No unicode bullets (`▸`, `•`), no `>` blockquote item headers, no `«»` placeholders.
- **One `#` title and one description paragraph per source `.md`** so the retriever can score each chunk in a single read.
- **Inline `Key: value | Key: value` stat lines** for token efficiency.
- **Markdown tables** for repeated structure (drops, recipes, comparisons), with `\|` escapes inside cells.
- **Slot tokens use `[NAME]`** (square brackets) — Gemma is more reliable than with `«PLACEHOLDER»`.
- **Aggressive wiki cleanup.** Sections such as Patch History, Gallery, Maximization, and See Also are stripped upstream by `cleanWiki()`.
- **System prompt is delivered as the FIRST user turn** by the bot, since Gemma has no system role. The prompt lives in the upstream repo at `prompts/oda-fragment.md`.
- **Retrieval-only context.** Gemma cannot hold all ~14 MB at once; the bot's retriever injects only the relevant slice of the bundle into each user turn.

## For AI agents reading this repo

- Treat `data/warframe-data-*.txt` as **read-only generated artifacts**. To change their content, edit the upstream pipeline (`pull.js` / `extract-wiki.py` / `transform.js` / `organize-docs.js`) in [`aidenlong04/warframe-item-pull`](https://github.com/aidenlong04/warframe-item-pull) and let the publish workflow re-emit them here.
- The current snapshot's provenance (source commit, generation time, workflow run URL) is in `data/MANIFEST.md`.
- Bundle filenames are stable; group bundles by prefix:
  - `warframe-data-drops-*` for reward tables,
  - `warframe-data-export*` for DE public-export-derived data,
  - everything else is grouped by domain (equipment, weapons, mods, items, lore, quests, mechanics, systems, patchnotes, mastery-rank).
- No build, lint, or test tooling is configured in this repo; there is nothing to run here.
